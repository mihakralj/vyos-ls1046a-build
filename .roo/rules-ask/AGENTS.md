# Ask Mode Rules (Non-Obvious Only)

- **`PORTING.md`** is the canonical technical reference for WHY each kernel config exists — not just what it does
- **`UBOOT.md`** contains U-Boot memory map, all boot commands (working + failed), and hardware details — this is the "what we tried" log
- **`INSTALL.md`** is the user-facing install guide — step-by-step from SSH to first login
- **`CHANGELOG.md`** is auto-generated by CI from upstream vyos-1x and vyos-build PRs — do not hand-edit
- **Two completely different boot flows exist:** (1) `dd` live-boot squashfs+overlay on `mmcblk0p2`, (2) `install image` creates GPT with BIOS boot + EFI + root partitions — different U-Boot commands, different partition layouts
- **The DTB story is complex:** `mono-gw.dtb` is extracted from live OpenWrt (NXP SDK kernel), contains SDK-specific nodes NOT recognized by mainline. `fsl-ls1046a-rdb.dtb` is compiled from mainline kernel source during build. If networking fails with mono-gw.dtb, test with RDB DTB to isolate the issue.
- **"Generic ARM64 VyOS" won't work here:** Three things are missing — eMMC driver, DPAA1 networking stack, and correct serial console. All are kernel config, documented in `PORTING.md`
- **FMan firmware is NOT a Linux firmware file:** U-Boot injects it from SPI flash into the DTB before kernel handoff. No `/lib/firmware/` files needed.
